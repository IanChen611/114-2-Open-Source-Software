# Kernel Module（sudoku.ko）

## 前置作業：取得 WSL2 Kernel 原始碼並編譯（僅需做一次）

建議先使用專案提供的自動準備流程：

```bash
make prepare-wsl2-kernel
```

此目標會依照 `uname -r` 推導 Microsoft WSL2 kernel branch。
例如 `6.18.33.1-microsoft-standard-WSL2` 會使用
`linux-msft-wsl-6.18.y`。
腳本會在建置前檢查 `make kernelrelease` 是否等於 `uname -r`；
若版本不一致，請改用符合目前 WSL2 kernel 的 branch、tag 或 commit。

若需要手動指定 branch：

```bash
WSL_BRANCH=linux-msft-wsl-6.18.y make prepare-wsl2-kernel
```

若 branch 最新提交與目前執行中的 WSL2 kernel patch 版本不一致，可指定
對應 commit：

```bash
WSL_BRANCH=linux-msft-wsl-6.18.y WSL_COMMIT=<commit> make prepare-wsl2-kernel
```

手動流程如下：

```bash
# 下載對應版本的 WSL2 kernel source
git clone --depth=1 --branch linux-msft-wsl-6.18.y \
    https://github.com/microsoft/WSL2-Linux-Kernel.git ~/WSL2-Linux-Kernel

# 安裝編譯依賴
sudo apt install build-essential flex bison libssl-dev libelf-dev bc dwarves

# 套用目前執行中的 kernel config
cd ~/WSL2-Linux-Kernel
cp /proc/config.gz . && gunzip config.gz && mv config .config
make olddefconfig

# 完整編譯（約 30–60 分鐘，產生 Module.symvers）
make -j$(nproc)
```

## 編譯 Kernel Module

```bash
# 回到專案目錄
cd /path/to/114-2-Open-Source-Software

make
```

> 若出現 `Module.symvers is missing` 警告但 kernel source 尚未完整編譯完成，
> 可暫時使用以下指令跳過 symbol 版本驗證：
> ```bash
> KBUILD_MODPOST_WARN=1 make
> ```

## 載入 Kernel Module

```bash
sudo insmod sudoku_module.ko

# 確認 /dev/sudoku 已出現
ls /dev/sudoku

# 查看 kernel log 確認載入成功
dmesg | tail -5
```

## 啟動 CLI 遊戲

```bash
# 編譯 CLI 客戶端（只需做一次）
make userspace

# 啟動遊戲
sudo ./sudoku-cli
```

### 遊戲指令

| 指令 | 說明 |
|---|---|
| `列 欄 數字` | 填入數字（1–9 為 1-indexed；0 代表清除） |
| `hint` | 自動填入一格正確答案 |
| `reset` | 還原棋盤至初始謎題 |
| `new` | 開始新遊戲 |
| `quit` | 離開程式 |

## 卸載 Kernel Module

```bash
sudo rmmod sudoku_module

# 確認已卸載
dmesg | tail -3
```

# WSL2 整合測試

在乾淨的 WSL2 Ubuntu 環境中，先依照上方步驟準備好
`~/WSL2-Linux-Kernel` 或 `/lib/modules/$(uname -r)/build`，
再於專案根目錄執行：

```bash
make integration-test
```

此目標會自動執行：

1. `make userspace`
2. `make`
3. `sudo insmod sudoku_module.ko`
4. 檢查 `/dev/sudoku`
5. 讀取 82 bytes 棋盤狀態，確認生成題目剛好移除 40 格
6. 透過 `sudoku-cli` 測試 `hint`、`reset`、`new`
7. `sudo rmmod sudoku_module`

成功時會輸出：

```text
WSL2 integration test passed.
```

同時會在專案根目錄產生 `integration-test.log`，可作為整合測試紀錄。

## 清除編譯產物

```bash
make clean
```
